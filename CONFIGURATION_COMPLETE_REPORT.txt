====================================================================================================
                                技术实现说明
====================================================================================================

工作流程:
  1. Agent启动时调用 load_tools_from_db()
  2. 从 agent_tools 表读取配置
  3. ToolConverter 将4种类型转换为统一的OpenAI格式
  4. LLM收到工具列表，根据用户问题决定调用哪个工具
  5. ToolRouter 解析工具名称，路由到对应的执行器
  6. ToolExecutor 执行具体的工具（Plugin/MCP/Function/Skill）
  7. 结果返回给LLM，生成最终回复

优先级系统:
  • 数字越大优先级越高
  • MCP (20-18) > Function (17-15) > Plugin (14-12) > Skill (11)
  • 当多个工具都适用时，LLM会倾向选择优先级高的

工具命名规范:
  • MCP:      mcp_{mcp_id}_{tool_name}
  • Function: function_{function_id}
  • Plugin:   plugin_{plugin_id}
  • Skill:    skill_{skill_id}
